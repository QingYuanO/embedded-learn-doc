> 面向嵌入式入门。读数据手册、写寄存器配置、算定时器参数时，单位搞错是最常见的低级失误之一。  
> 本文按类别梳理嵌入式开发中**高频出现的单位**，并说明各自**代表什么、在哪里用、怎么换算**。


## 一、国际单位词头

嵌入式文档和代码里大量出现带前缀的单位，先记住这张表：

| 词头 | 符号 | 倍数 | 示例 |
|------|------|------|------|
| 吉 | G | 10⁹ | GHz、GB |
| 兆 | M | 10⁶ | MHz、MB、MΩ |
| 千 | k | 10³ | kHz、kΩ、KB |
| — | — | 1 | Hz、V、A |
| 毫 | m | 10⁻³ | ms、mV、mA |
| 微 | μ / u | 10⁻⁶ | μs、μF（代码常写 `uF`） |
| 纳 | n | 10⁻⁹ | ns、nF |
| 皮 | p | 10⁻¹² | pF |

**记忆口诀**：大写 **M**（兆）和小写 **m**（毫）差 10⁹ 倍，写错数量级电路直接废掉。

```
1 MHz = 1,000 kHz = 1,000,000 Hz
1 ms  = 1,000 μs = 1,000,000 ns
1 mA  = 0.001 A
```

## 二、频率与时间

### 2.1 频率：Hz

**Hz（赫兹）** = 每秒发生多少次。

| 单位 | 含义 | 嵌入式场景 |
|------|------|------------|
| **Hz** | 次/秒 | PWM 频率、步进脉冲频率、中断节拍 |
| **kHz** | 千次/秒 | 定时器计数频率、音频采样率 |
| **MHz** | 百万次/秒 | MCU 主频、外设时钟 |

典型例子：

- STM32F103 主频 **72 MHz** → CPU 每秒执行约 7200 万条指令周期（实际指令数还取决于流水线）
- TIM 预分频后计数频率 **10 kHz** → 每 0.1 ms 计数加 1
- 步进电机 **400 Hz** STEP 脉冲 → 每秒 400 微步

### 2.2 时间：s、ms、μs、ns

| 单位 | 含义 | 嵌入式场景 |
|------|------|------------|
| **s** | 秒 | 超时、看门狗、长延时 |
| **ms** | 毫秒 | `Delay_ms()`、RTOS tick、按键消抖 |
| **μs** | 微秒 | SPI/I2C 时序、STEP 最小脉宽、输入捕获 |
| **ns** | 纳秒 | 信号上升沿、高速 IO 切换时间 |

**频率与时间互为倒数**（周期 T）：

```
T = 1 / f

例：f = 1 kHz  →  T = 1 ms
    f = 1 MHz  →  T = 1 μs
```

### 2.3 定时器里的「时钟」与「频率」

配置 TIM 时常见两个量：

```
TIM 计数时钟 = 系统时钟 / (PSC + 1)
中断频率     = TIM 计数时钟 / (ARR + 1)
```

例：72 MHz 系统时钟，`PSC = 7200 - 1`，`ARR = 100 - 1`：

```
计数时钟 = 72MHz / 7200 = 10 kHz
中断频率 = 10kHz / 100 = 100 Hz  →  每 10 ms 进一次中断
```

## 三、数据与存储

### 3.1 bit 与 Byte

| 单位 | 含义 | 说明 |
|------|------|------|
| **bit（位）** | 0 或 1 | 寄存器某一位、GPIO 电平 |
| **Byte（字节）** | 8 bit | `uint8_t`、Flash 存储基本单位 |
| **Word（字）** | 通常 32 bit | ARM Cortex-M 一次处理宽度 |
| **Half-word** | 16 bit | `uint16_t`、STM32 半字访问 |

```
1 Byte = 8 bit
1 KB   = 1024 Byte（嵌入式语境下常按 1024 算，见下文 KiB）
1 MB   = 1024 KB
```

### 3.2 存储容量：KB / MB 与 KiB / MiB

| 写法 | 实际含义 | 常见出处 |
|------|----------|----------|
| **KB、MB** | 有时指 1000，有时指 1024 | 老文档、厂商标称 |
| **KiB、MiB** | 严格 1024 进制 | 规范写法 |

STM32F103C8 标称 **64 KB Flash、20 KB RAM**，在 ST 文档里通常按 **1024 进制**理解：

```
64 KB Flash ≈ 64 × 1024 = 65,536 Byte
```

### 3.3 带宽与吞吐量

| 单位 | 含义 | 场景 |
|------|------|------|
| **bps** | bit per second | 串口、CAN 理论速率 |
| **B/s** | Byte per second | DMA 搬运、SD 卡读写 |

```
1 Byte/s = 8 bps
115200 bps 串口，8N1 无压缩时，净载荷约 115200/10 ≈ 11.5 KB/s
（每字节 1 起始位 + 8 数据位 + 1 停止位 = 10 bit）
```

### 3.4 地址与指针对齐

- **地址**：无单位数字，表示内存或外设寄存器位置，如 `0x40010800`（GPIOA 基址）
- **对齐**：ARM 要求 32 位访问地址 4 字节对齐，否则 HardFault

## 四、电气量

### 4.1 电压 V、mV

| 单位 | 场景 |
|------|------|
| **V** | 电源 3.3 V / 5 V / 12 V、ADC 输入范围 |
| **mV** | 传感器输出、运放偏置、ADC 分辨率换算 |

```
ADC 数字量 → 电压：
V = (adc_value / ADC_MAX) × Vref

例：12 位 ADC，Vref = 3.3 V，读数 2048：
V = 2048 / 4096 × 3.3 ≈ 1.65 V
```

### 4.2 电流 A、mA、μA

| 单位 | 场景 |
|------|------|
| **A** | 电机、电源模块额定 |
| **mA** | GPIO 驱动能力、LED 限流、LDO 输出 |
| **μA** | 休眠电流、RTC 备份域 |

单片机 GPIO 通常只能提供 **几 mA～25 mA** 级别，大电流负载必须加驱动芯片。

### 4.3 电阻 Ω、kΩ、MΩ

| 单位 | 场景 |
|------|------|
| **Ω** | 限流电阻、采样电阻 |
| **kΩ** | 上拉/下拉、分压 |
| **MΩ** | 高阻输入、漏电路径 |

LED 限流估算：

```
R ≈ (Vcc - Vf) / If

例：3.3 V 供电，LED 压降 2.0 V，目标 10 mA：
R ≈ (3.3 - 2.0) / 0.01 ≈ 130 Ω → 取标称 150 Ω
```

### 4.4 电容 F、μF、nF、pF

| 单位 | 场景 |
|------|------|
| **F** | 理论单位，实际很少直接用 |
| **μF** | 电源 bulk 电容、电解 |
| **nF** | 去耦、滤波 |
| **pF** | 晶振负载电容、高频旁路 |

MCU 电源脚典型去耦：**100 nF（0805/0603）+ 10 μF**，尽量靠近芯片。

### 4.5 功率 W、mW

```
P = V × I

例：3.3 V、50 mA 工作电流 → P = 3.3 × 0.05 ≈ 0.165 W
```

用于估算 LDO 发热、MOSFET 功耗、电机驱动散热需求。

## 五、通信速率

### 5.1 波特率 Baud

**波特率** = 线路上每秒变化的符号数。UART 常用 **115200、9600** 等。

配置 USART 时：

```c
USART_InitStructure.USART_BaudRate = 115200;  /* 115200 符号/秒 */
```

与 **bps（bit/s）** 在 UART 二进制场景下数值相同，但概念上波特率强调「符号」，多电平调制时二者可能不同。

### 5.2 SPI / I2C 时钟

| 接口 | 单位 | 典型值 |
|------|------|--------|
| **SPI SCK** | Hz | 1 MHz～几十 MHz |
| **I2C SCL** | Hz | 100 kHz（标准）、400 kHz（快速） |

MS41949 手册要求 SCK **最高约 5 MHz**；I2C 设备常见 **400 kHz** 上限。

### 5.3 CAN 比特率

常见 **125 kbps、250 kbps、500 kbps、1 Mbps**，单位是 **bit/s**。

## 六、角度与机械转速

### 6.1 角度

| 单位 | 含义 | 场景 |
|------|------|------|
| **°（度）** | 360° 为一圈 | 步进电机 1.8°/步、舵机角度 |
| **rad（弧度）** | 2π rad = 360° | 算法、三角函数 `sin()` |

```
rad = deg × π / 180
```

### 6.2 转速 RPM

**RPM** = Revolutions Per Minute，**转/分钟**。

```
RPM = (步频 / 每圈步数) × 60

例：1.8° 电机，16 细分 → 每圈 3200 微步，步频 1600 Hz：
RPM = 1600 / 3200 × 60 = 30 转/分钟
```

### 6.3 脉冲当量 pps

**pps** = pulses per second，**脉冲/秒**，常见于步进驱动、编码器说明，与 **Hz** 数值相同，强调「步进脉冲」语义。

## 七、温度

| 单位 | 场景 |
|------|------|
| **°C** | 环境温度、芯片结温、传感器读数 |
| **K（开尔文）** | 热力学计算，K = °C + 273.15 |

MCU 数据手册 **工作温度** 如 `-40 °C ~ +85 °C`（工业级）或 `0 °C ~ +70 °C`（商业级）。

数字温度传感器（如 LM75、DS18B20）直接输出摄氏度或需按手册公式换算。

## 八、ADC 与测量

### 8.1 分辨率：bit 与 LSB

| 术语 | 含义 |
|------|------|
| **N bit ADC** | 输出 0～2^N - 1 的数字量 |
| **LSB** | Least Significant Bit，最小量化步长 |

12 位 ADC → 0～4095，**1 LSB 对应的电压**：

```
V_LSB = Vref / 4096

Vref = 3.3 V 时，V_LSB ≈ 0.806 mV
```

### 8.2 采样率

**采样率** 单位 **Hz** 或 **kSPS**（kilo Samples Per Second）。

```
Nyquist：采样率至少为被测信号最高频率的 2 倍
例：测 0～5 kHz 音频 → 采样率 ≥ 10 kHz，常用 8～48 kHz
```

### 8.3 百分比与占空比

**PWM 占空比** = 高电平时间 / 周期 × 100%

```
占空比 = CCR / (ARR + 1) × 100%

例：ARR = 999，CCR = 250 → 占空比 25%
```

## 九、C 代码中的单位写法

### 9.1 整型后缀

| 后缀 | 含义 | 示例 |
|------|------|------|
| **u / U** | unsigned | `72000000u` |
| **l / L** | long | `100000L` |
| **ul / UL** | unsigned long | `27000000UL` |
| **ll / LL** | long long | 大计数 |

定时、频率计算推荐加 **`u` 后缀**，避免有符号溢出：

```c
#define SYSCLK_HZ  72000000u
#define PSC        7200u
uint32_t tim_clk = SYSCLK_HZ / PSC;  /* 10 kHz */
```

### 9.2 浮点后缀

| 后缀 | 含义 |
|------|------|
| **f / F** | float | `3.3f` |
| 无后缀 | double（默认） |

嵌入式为省 Flash/RAM，三角函数等常显式用 `float`：

```c
float v = (float)adc * 3.3f / 4096.0f;
```

### 9.3 进制前缀

| 前缀 | 含义 | 示例 |
|------|------|------|
| **0x** | 十六进制 | `0x40010800`（寄存器地址） |
| **0b** | 二进制（C23 / 编译器扩展） | `0b00001111` |
| **0** | 八进制 | 少用 |

位操作与寄存器配置几乎都用 **十六进制**，与数据手册一致。

### 9.4 宏封装带单位的常量

推荐把物理含义写进名字，避免裸数字：

```c
#define MS41949_OSCIN_HZ    27000000u
#define UART_BAUD           115200u
#define LED_GPIO_SPEED      GPIO_Speed_50MHz  /* ST 库枚举，非 SI 单位 */
```

## 十、常见踩坑

### 10.1 词头大小写写反

```c
/* 错：想要 1 毫秒，写成了 1000 秒量级 */
Delay_ms(1000 * 1000);   /* 若本意是 us 级，单位搞混 */

/* 错：7200M 当 7200 MHz */
7200000   /* 这是 7.2 MHz，不是 7200 MHz */
```

### 10.2 整数除法丢精度

```c
/* 错：结果为 0 */
uint32_t arr = 10000 / 3000;

/* 对：先乘后除，或转 float */
uint32_t arr = 10000u / 3000u;  /* 仍得 3，需明确舍入策略 */
uint32_t arr = (10000u + step_hz/2) / step_hz;  /* 四舍五入示例 */
```

### 10.3 溢出

```c
/* 错：16 位乘法溢出 */
uint16_t a = 10000, b = 10000;

/* 对：提升到 32 位 */
uint32_t c = (uint32_t)a * b;
```

72 MHz × 768 这类计算，中间结果务必用 **uint32_t 或 uint64_t**。

### 10.4 KB 与 kbit 混淆

下载速度说 **128 KB/s** 和 **128 kbps** 差 8 倍；串口 **115200 bps** 不是 115200 Byte/s。

### 10.5 时间单位混用

数据手册写 **minimum 1 μs pulse width**，代码里用 `Delay_ms(1)` 等于 1000 μs，差三个数量级。

## 总结

| 要点 | 说明 |
|------|------|
| 词头 | M（兆）与 m（毫）差 10⁹，写代码前先对数量级 |
| 频率 ↔ 时间 | `T = 1/f`，定时器参数都从这里来 |
| 存储 | 1 Byte = 8 bit；Flash/RAM 常用 1024 进制 |
| 电气 | V、A、Ω、F 配单位换算，LED/ADC/LDO 都要算 |
| 通信 | baud、bps、SPI Hz 别混 |
| 代码 | 频率/地址用 `u` 后缀和宏命名，防溢出 |

单位本身不难，难在**读手册、写代码、看波形**时始终带着量纲。养成习惯：**每个数字都问一句「这是什么单位？」**，能少踩一半入门坑。
