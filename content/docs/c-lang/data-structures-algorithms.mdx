> 面向嵌入式入门。MCU 上 RAM 有限、很少用 `malloc`，但 **UART 缓冲、任务队列、传感器采样排序、寄存器表查找** 都离不开数据结构与基础算法。  
> 本文从 **数组、链表、栈、队列** 讲到 **线性/二分查找** 与 **冒泡、选择、插入、快排** 等常用排序，并给出 C 实现与嵌入式选型建议。


## 一、先建立两个概念

### 1.1 数据结构 vs 算法

| 概念 | 含义 | 例子 |
|------|------|------|
| **数据结构** | 数据在内存中的组织方式 | 数组、链表、队列 |
| **算法** | 对数据的操作步骤 | 查找、排序、入队出队 |

同一批数据，用数组存还是链表存，会直接影响 **查找快慢、插入是否搬移、占多少 RAM**。

### 1.2 时间复杂度（Big-O）

只关心 **数据量 n 变大时，操作次数如何增长**，忽略常数和低阶项。

| 记号 | 含义 | 嵌入式直觉 |
|------|------|------------|
| **O(1)** | 与 n 无关 | 最理想，寄存器读写 |
| **O(log n)** | 对数增长 | 很好，有序数组二分 |
| **O(n)** | 线性 | 可接受，遍历一次 |
| **O(n log n)** | 常见高效排序 | 中等数据量可用 |
| **O(n²)** | 平方 | 小 n（几十）尚可，上千慎用 |

## 二、数组：最基础、最高效

**数组**在内存中 **连续** 存放同类型元素，通过下标 O(1) 随机访问。

```c
#define N  16

int16_t samples[N];   /* 16 个 int16，连续占 32 字节 */

samples[0] = 100;
samples[5] = *p;        /* 下标访问，编译器算 offset = 5 * sizeof(int16_t) */
```

```
地址低 ────────────────────────────── 地址高
[s0][s1][s2][s3] ... [s15]
 ↑
 samples 名即首元素地址（退化为指针）
```

| 优点 | 缺点 |
|------|------|
| 随机访问 O(1) | 长度固定（或需 realloc） |
| Cache 友好、无额外指针开销 | 中间插入/删除要搬移 O(n) |
| 嵌入式最常用 | 过大数组占 RAM/Flash |

**典型用途**：ADC 采样缓冲、查找表（`const` 放 Flash）、FFT 输入、排序工作区。

---

## 三、链表：指针把节点串起来

### 3.1 单链表

```c
typedef struct Node
{
    int32_t         data;
    struct Node    *next;
} Node;
```

```
head → [d|next] → [d|next] → [d|NULL]
```

**头插** O(1)：

```c
void list_push_front(Node **head, int32_t val, Node *pool, uint16_t *pool_idx)
{
    Node *n = &pool[(*pool_idx)++];   /* 嵌入式常用静态池代替 malloc */
    n->data = val;
    n->next = *head;
    *head = n;
}
```

**按值查找** O(n)：从 `head` 沿 `next` 走到 `NULL`。

**删除指定节点** 需找前驱，单链 O(n)；若只有头指针、删中间节点较麻烦。

### 3.2 双链表

每个节点多一个 `prev`，可双向遍历、O(1) 删已知节点（有前驱指针时）。

```c
typedef struct DNode
{
    int32_t         data;
    struct DNode   *prev;
    struct DNode   *next;
} DNode;
```

RAM 开销更大，嵌入式 **不如单链 + 环形队列常见**。

### 3.3 链表 vs 数组

| 操作 | 数组 | 单链表 |
|------|------|--------|
| 按下标访问 | O(1) | 不支持 |
| 头插 | O(n) 搬移 | O(1) |
| 查找 | O(n) | O(n) |
| 内存 | 连续、紧凑 | 每节点多指针，可能不连续 |
| malloc | 不需要 | 常需池化或 malloc |

**选型**：长度固定、要排序/二分 → **数组**；长度变化、频繁头尾增删 → **链表或环形队列**。

## 四、栈（Stack）：后进先出 LIFO

**栈**只允许在一端 **压入 push** 和 **弹出 pop**，像叠盘子。

```
        push ↓
    ┌───┬───┬───┐
    │ C │ B │ A │  ← top
    └───┴───┴───┘
        pop  ↑  先弹出 C
```

### 4.1 数组实现（推荐）

```c
#define STACK_CAP  32

typedef struct
{
    int32_t  buf[STACK_CAP];
    uint16_t top;    /* 指向下一个空位；0 表示空栈 */
} Stack;

void stack_init(Stack *s)
{
    s->top = 0;
}

int stack_push(Stack *s, int32_t v)
{
    if (s->top >= STACK_CAP)
        return -1;           /* 栈满 */
    s->buf[s->top++] = v;
    return 0;
}

int stack_pop(Stack *s, int32_t *out)
{
    if (s->top == 0)
        return -1;           /* 栈空 */
    *out = s->buf[--s->top];
    return 0;
}
```

### 4.2 程序栈（调用栈）

函数调用时，局部变量和返回地址压在 **CPU 栈**（`memory-layout` 一文中的 stack 区）。递归过深 → **栈溢出**。这是「栈」这个词在嵌入式里的另一层含义，与上面的数据结构栈概念相关但物理上是同一块 RAM 向下增长。

**典型用途**：表达式求值、DFS 深度优先、括号匹配、中断嵌套上下文（硬件栈）。

## 五、队列（Queue）：先进先出 FIFO

**队列**从 **rear 入队**、**front 出队**，像排队。

### 5.1 环形数组队列（嵌入式首选）

```c
#define Q_CAP  64

typedef struct
{
    uint8_t   buf[Q_CAP];
    uint16_t  head;    /* 出队位置 */
    uint16_t  tail;    /* 入队位置 */
    uint16_t  count;   /* 当前元素个数 */
} RingQueue;

void rq_init(RingQueue *q)
{
    q->head = q->tail = q->count = 0;
}

int rq_push(RingQueue *q, uint8_t byte)
{
    if (q->count >= Q_CAP)
        return -1;
    q->buf[q->tail] = byte;
    q->tail = (q->tail + 1) % Q_CAP;
    q->count++;
    return 0;
}

int rq_pop(RingQueue *q, uint8_t *out)
{
    if (q->count == 0)
        return -1;
    *out = q->buf[q->head];
    q->head = (q->head + 1) % Q_CAP;
    q->count--;
    return 0;
}
```

```
       tail 入队
         ↓
  ... [x][x][·][·][x][x] ...
         ↑           ↑
       head 出队   数组末尾绕回 0（取模）
```

**典型用途**：UART 收发缓冲、按键事件、FreeRTOS 消息队列底层、采样流水线。

### 5.2 链表队列

维护 `front` 和 `rear`，入队尾插 O(1)，出队头删 O(1)。适合长度变化大、但需内存池；固定缓冲场景 **环形数组更简单**。

---

## 六、其他常见结构（简要）

| 结构 | 特点 | 嵌入式场景 |
|------|------|------------|
| **哈希表** | 平均 O(1) 查找，需额外空间与哈希函数 | 协议 ID → 处理函数映射（小表可数组代替） |
| **二叉搜索树 BST** | 有序插入，平均 O(log n) 查找 | 动态键值，MCU 上较少 |
| **堆（优先队列）** | 取最值 O(log n) | 定时器调度、Top-K |
| **位图 / 位域** | 极省 RAM | GPIO 状态、标志位集合 |

小 MCU 上 **90% 是数组 + 环形队列 + 偶尔链表**；复杂结构多在 Linux 侧或外部 RAM 芯片上。

## 七、查找算法

### 7.1 线性查找（顺序查找）

**无序**数组从头到尾比，找到即返回下标，否则 -1。

```c
int linear_search(const int32_t *a, uint16_t n, int32_t key)
{
    uint16_t i;

    for (i = 0; i < n; i++)
    {
        if (a[i] == key)
            return (int)i;
    }
    return -1;
}
```

| 项目 | 值 |
|------|-----|
| 前提 | 无顺序要求 |
| 时间 | O(n) |
| 空间 | O(1) |
| 适用 | n 小（几十）、只查一次、表在 Flash |

**例子**：从 20 个 `const` 引脚配置里找匹配项；中断里查短表。

### 7.2 二分查找（折半查找）

**前提**：数组必须 **已按升序（或降序）排好**。

思路：每次比较中间元素，缩小一半范围。

```c
int binary_search(const int32_t *a, uint16_t n, int32_t key)
{
    uint16_t lo = 0;
    uint16_t hi = n;       /* 左闭右开 [lo, hi) */
    uint16_t mid;

    while (lo < hi)
    {
        mid = lo + (hi - lo) / 2;   /* 防 (lo+hi) 溢出，写法更稳 */

        if (a[mid] == key)
            return (int)mid;
        if (a[mid] < key)
            lo = mid + 1;
        else
            hi = mid;
    }
    return -1;
}
```

```
key = 23，有序数组 [3, 7, 11, 23, 31, 42]

第 1 次：mid → 11，23 > 11，搜右半
第 2 次：mid → 31，23 < 31，搜左半
第 3 次：mid → 23，命中
```

| 项目 | 值 |
|------|-----|
| 时间 | O(log n) |
| 空间 | O(1) 迭代版 |
| 适用 | 有序表、多次查找、n 较大 |

**注意**：

- 必须先排序，或表本身静态有序（如频率表）。
- 不适合链表（无法 O(1) 取中点，除非加长度且走指针 O(n)）。

### 7.3 查表（LUT）

嵌入式极常见：**数组即表**，下标或线性扫描。

```c
static const uint32_t baud_table[] = { 9600, 19200, 38400, 115200 };

uint32_t baud = baud_table[index];   /* O(1) 直接索引 */
```

若键不是连续下标，可用 **有序键值对 + 二分**，或小规模 **线性查**。

### 7.4 查找算法对比

| 算法 | 前提 | 时间 | 嵌入式建议 |
|------|------|------|------------|
| 线性查找 | 无 | O(n) | n ≤ 32~64 首选 |
| 二分查找 | 有序 | O(log n) | 有序大表、多次查 |
| 哈希 | 哈希函数 | 平均 O(1) | 键多、查频繁、有 RAM |
| 查表 | 下标映射 | O(1) | 档位、枚举、波形表 |

## 八、排序算法

排序把元素变成 **升序或降序**。以下均按 **升序、整型数组** 讲解，嵌入式可原地排序省 RAM。

### 8.1 评价维度

| 维度 | 说明 |
|------|------|
| **时间** | 平均 / 最坏 |
| **空间** | 是否需要与 n 同量级的辅助数组 |
| **稳定性** | 相等元素排序后相对顺序是否不变 |
| **原地** | 是否只用 O(1) 额外空间 |

稳定排序对「先按优先级再按时间」的多键记录有时重要。

---

### 8.2 冒泡排序（Bubble Sort）

相邻两两比较，大者往后「冒泡」，每轮把最大值沉到末尾。

```c
void bubble_sort(int32_t *a, uint16_t n)
{
    uint16_t i, j;
    int32_t  tmp;
    uint8_t  swapped;

    for (i = 0; i + 1 < n; i++)
    {
        swapped = 0;
        for (j = 0; j + 1 < n - i; j++)
        {
            if (a[j] > a[j + 1])
            {
                tmp = a[j];
                a[j] = a[j + 1];
                a[j + 1] = tmp;
                swapped = 1;
            }
        }
        if (!swapped)    /* 已有序则提前结束 */
            break;
    }
}
```

| 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|
| O(n²) | O(n²) | O(1) | 是 |

**嵌入式**：代码极简，**n ≤ 20~30** 可接受；教学、偶尔整理小缓冲可用。

---

### 8.3 选择排序（Selection Sort）

每轮在未排序段选 **最小值**，放到已排序段末尾。

```c
void selection_sort(int32_t *a, uint16_t n)
{
    uint16_t i, j, min_idx;
    int32_t  tmp;

    for (i = 0; i + 1 < n; i++)
    {
        min_idx = i;
        for (j = i + 1; j < n; j++)
        {
            if (a[j] < a[min_idx])
                min_idx = j;
        }
        if (min_idx != i)
        {
            tmp = a[i];
            a[i] = a[min_idx];
            a[min_idx] = tmp;
        }
    }
}
```

| 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|
| O(n²) | O(n²) | O(1) | 否 |

交换次数 ≤ n-1，**写 Flash/RAM 少**，但比较次数仍是 O(n²)。小数组、写敏感时可选。

---

### 8.4 插入排序（Insertion Sort）

把未排序元素逐个 **插入** 到已排序段正确位置；对已近乎有序的数组很快。

```c
void insertion_sort(int32_t *a, uint16_t n)
{
    uint16_t i, j;
    int32_t  key;

    for (i = 1; i < n; i++)
    {
        key = a[i];
        j = i;
        while (j > 0 && a[j - 1] > key)
        {
            a[j] = a[j - 1];
            j--;
        }
        a[j] = key;
    }
}
```

| 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|
| O(n²) | O(n²) | O(1) | 是 |

**近乎有序** 时接近 O(n)。在线算法：每来一个新采样插入有序区，适合 **流式小集合**。

---

### 8.5 快速排序（Quick Sort）

选 **基准 pivot**（常取最右或中间），分区：小于 pivot 放左，大于放右，再递归左右。

```c
static void swap32(int32_t *x, int32_t *y)
{
    int32_t t = *x;
    *x = *y;
    *y = t;
}

static uint16_t partition(int32_t *a, uint16_t lo, uint16_t hi)
{
    int32_t  pivot = a[hi];
    uint16_t i = lo;
    uint16_t j;

    for (j = lo; j < hi; j++)
    {
        if (a[j] <= pivot)
        {
            swap32(&a[i], &a[j]);
            i++;
        }
    }
    swap32(&a[i], &a[hi]);
    return i;
}

static void quick_sort_rec(int32_t *a, uint16_t lo, uint16_t hi)
{
    uint16_t p;

    if (lo >= hi)
        return;
    p = partition(a, lo, hi);
    if (p > 0)
        quick_sort_rec(a, lo, p - 1);
    quick_sort_rec(a, p + 1, hi);
}

void quick_sort(int32_t *a, uint16_t n)
{
    if (n > 0)
        quick_sort_rec(a, 0, n - 1);
}
```

| 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|
| O(n log n) | O(n²) pivot 不佳 | O(log n) 栈深度 | 否 |

**嵌入式注意**：

- **递归** 占用调用栈；n 大时改 **迭代 + 显式栈**，或限制递归深度。
- 最坏 O(n²) 可通过 **三数取中** 选 pivot 缓解。
- 中等数组（几百个 int）在 RAM 够用时 **性能首选**。

---

### 8.6 归并排序（Merge Sort）

分治：拆两半 → 排序 → **合并** 两个有序段。需要 **O(n) 辅助数组**。

| 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|
| O(n log n) | O(n log n) | O(n) | 是 |

时间稳定可预期，但 **多占 n 个元素 RAM**，20 KB SRAM 的 MCU 上要算账。外部 RAM 或 PC 侧常用。

---

### 8.7 堆排序（Heap Sort）

建堆后反复取堆顶。原地 O(n log n)，不稳定。

MCU 上若已有 **优先队列** 需求可一并了解；仅为排序时，快排往往更简单。

---

### 8.8 排序算法总览

| 算法 | 平均时间 | 最坏 | 额外空间 | 稳定 | 嵌入式小 n | 嵌入式大 n |
|------|----------|------|----------|------|------------|------------|
| 冒泡 | O(n²) | O(n²) | O(1) | 是 | 可用 | 不推荐 |
| 选择 | O(n²) | O(n²) | O(1) | 否 | 可用 | 不推荐 |
| 插入 | O(n²) | O(n²) | O(1) | 是 | 推荐 | 不推荐 |
| 快速 | O(n log n) | O(n²) | O(log n) 栈 | 否 | 可以 | **推荐**（防栈溢出） |
| 归并 | O(n log n) | O(n log n) | O(n) | 是 | 可以 | RAM 够才用 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 否 | 少用 | 可选 |

**C 标准库**：`qsort()`（stdlib.h）可调用，需比较函数指针，有少量栈/代码体积开销：

```c
#include <stdlib.h>

static int cmp_int(const void *pa, const void *pb)
{
    int32_t a = *(const int32_t *)pa;
    int32_t b = *(const int32_t *)pb;
    return (a > b) - (a < b);
}

qsort(arr, n, sizeof(arr[0]), cmp_int);
```

链接/newlib 可能带入较多代码；极小工程可 **手写插入排序** 控制体积。

## 十、嵌入式选型 checklist

1. **数据量 n 多少？** n ≤ 30 随便 O(n²)；上百考虑快排或 `qsort`。
2. **RAM 预算？** 归并要 O(n) 辅助；链表要指针；优先 **静态数组**。
3. **是否中断上下文？** 中断里别做长排序；缓冲进队列，主循环再处理。
4. **是否多次查找？** 查一次线性即可；查多次先排序再二分。
5. **是否动态长度？** 环形队列 + 固定池；避免裸 `malloc`。
6. **代码体积？** 比 `qsort` 小的手写插入/选择；Release 开 `-Os`。
7. **稳定性？** 多键记录用插入/归并；一般传感器标量快排即可。

## 十一、常见错误

| 错误 | 后果 |
|------|------|
| 未排序就二分 | 结果错误 |
| 环形队列不设 `count` 或容量判满 | 覆盖未读数据 |
| 快排递归过深 | 栈溢出 HardFault |
| 链表 `next` 未初始化 | 遍历跑飞 |
| 排序比较函数与元素类型不一致 | 未定义行为 |
| 有符号/无符号混比 | 比较结果错 |
| 在中断里 O(n²) 排序 | 丢中断、实时性崩 |

## 十二、小结

| 主题 | 要点 |
|------|------|
| **数组** | 连续、O(1) 下标、嵌入式主力 |
| **链表** | 指针串联、头插 O(1)、查找 O(n) |
| **栈** | LIFO，数组实现简单 |
| **队列** | FIFO，**环形数组**最常用 |
| **线性查找** | 无前提 O(n)，小表够用 |
| **二分查找** | 有序 O(log n)，多次查值得先排序 |
| **冒泡/选择/插入** | O(n²)，小 n、代码简单 |
| **快速排序** | 平均 O(n log n)，注意递归栈 |

把 **「数据怎么存」** 和 **「要怎么查、怎么排」** 分开想，再按 **n 的大小、RAM、是否中断** 选型，是在 MCU 上用好数据结构与算法的核心。
